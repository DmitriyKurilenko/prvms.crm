# Аудит (журнал действий)

## Для чего эта страница

Здесь видно, кто и когда что-то менял в системе: создание сделок, изменение контактов, удаление записей, смену ролей и так далее. Полезно, чтобы разобраться «что случилось с этой сделкой» или проверить действия сотрудника.

## Где её открыть

В левом меню нажмите **«Аудит»** (значок списка с галочками). Раздел доступен владельцу и администратору.

## Что показывает страница

Таблица событий: **Дата**, **Действие** (создание, изменение, удаление), **Объект** (тип записи — сделка, контакт, документ), а также пользователь, выполнивший действие.

Сверху — фильтры:

- по **пользователю**;
- по **действию**;
- по **дате** (период).

Список листается постранично (10/20/50 строк).

## Пошагово

### Найти нужные события
1. Откройте **«Аудит»**.
2. Задайте фильтры (пользователь, действие, период).
3. Список обновится.

### Посмотреть, что именно изменилось
1. Кликните на строку события.
2. Появится таблица различий **«Поле / Было / Стало»** — видно, какие поля и как поменялись.

### Выгрузить журнал
Нажмите кнопку экспорта — журнал сохранится в **CSV** с учётом текущих фильтров.

## Типичные ситуации

- **«Почему у сделки пропала сумма?»** — найдите изменение сделки за нужную дату и откройте различия.
- **«Кто удалил контакт?»** — отфильтруйте по действию «удаление».
- **История по сотруднику** — отфильтруйте по пользователю.

## Если не получается

- **Нет раздела «Аудит»** — доступ только у владельца и администратора.
- **Список пустой** — либо изменяемых действий ещё не было, либо они отсеяны фильтрами. Сбросьте фильтры.
- **При клике нет деталей** — у некоторых событий нет сохранённых изменений (например, просмотр не записывается).
