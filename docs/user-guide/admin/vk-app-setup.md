# Настройка приложения ВКонтакте для платформы

Эта инструкция предназначена для **администратора платформы** (владельца инфраструктуры). Выполняется один раз для всей платформы.

## Шаги

1. Зайдите на https://vk.com/dev → «Мои приложения» → «Создать приложение».
2. Выберите тип **Standalone-приложение**.
   > Важно: именно standalone, а не «веб-сайт». Для standalone-приложений ВКонтакте поддерживает Implicit Flow — токен возвращается прямо в браузер без необходимости хранить `client_secret` на сервере.
3. Укажите название **PRVMS CRM** (или другое, если требуется).
4. После создания зайдите в **Настройки** приложения:
   - **Доверенные redirect URI**: добавьте `https://ваш-домен/oauth/vk/callback`
   - **Состояние приложения**: «Приложение включено и видно всем»
5. Скопируйте **ID приложения** (число в верхней части настроек).
6. Добавьте его в `.env` (или `.env.prod`) платформы:
   ```
   VK_APP_ID=12345678
   ```
7. Перезапустите backend-контейнеры:
   ```bash
   docker compose up -d --build web celery celery-beat
   ```

## Почему Standalone, а не «веб-сайт»

Для управления сообществом ВКонтакте требуется токен сообщества (community access token). Получить его можно только через Implicit Flow — токен возвращается в адресной строке браузера после знака `#`. Этот режим работает только со Standalone-приложениями и не требует никаких серверных секретов. Безопасность обеспечивается привязкой к доверенному redirect URI.

Это упрощает multi-tenant эксплуатацию: одно приложение PRVMS обслуживает все организации на платформе.
