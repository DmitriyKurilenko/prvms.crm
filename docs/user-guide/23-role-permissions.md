# Права ролей

## Для чего эта вкладка

Здесь настраивается, что каждая роль может делать с записями CRM: смотреть, создавать, редактировать, удалять. А также — **какие именно** записи видит сотрудник (все, только команды или только свои).

## Где её открыть

В левом меню **«Команда»** → вкладка **«Права ролей»**. Доступно только владельцу и администратору.

## Что показывает таблица

Каждая строка — это сочетание «роль + тип записи».

- **Роль** — Владелец, Администратор, Менеджер, Наблюдатель.
- **Сущность** — **Сделки**, **Контакты**, **Компании**, **Товары**, **Веб-формы**.
- **Просмотр** — можно ли видеть записи.
- **Создание** — можно ли создавать.
- **Редакт.** — можно ли изменять.
- **Удаление** — можно ли удалять.
- **Область** — какие записи видны: **Все**, **Команда** или **Только свои** (где сотрудник — ответственный).

## Пошагово

### Настроить права для роли
1. Найдите нужную строку (например, «Менеджер» + «Сделки»).
2. Поставьте или снимите галочки «Просмотр», «Создание», «Редакт.», «Удаление».
3. В столбце **«Область»** выберите, какие записи видит сотрудник.
4. Нажмите **«Сохранить»** в строке.

### Как логика работает сама
- Если снять **«Просмотр»**, автоматически снимаются остальные права.
- Если включить любое из прав изменения, **«Просмотр»** включится автоматически.

## Примеры

- **Менеджер видит только свои сделки** — в строке «Менеджер + Сделки» область **«Только свои»**.
- **Наблюдатель видит всё, но не меняет** — для каждой сущности только «Просмотр», область **«Все»**.
- **Запретить менеджерам удалять компании** — в строке «Менеджер + Компании» снимите «Удаление».

## Если не получается

- **Вкладка не открывается** — доступ только у владельца и администратора.
- **Галочки «прыгают» обратно** — система приводит права в согласованный вид (нельзя удалять то, что не видно). Пересохраните.
- **Права не применились к сотруднику** — попросите его выйти и войти заново.
