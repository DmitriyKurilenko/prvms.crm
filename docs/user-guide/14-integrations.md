# Интеграции с внешними CRM

## Для чего эта страница

Связать вашу систему с уже используемой CRM (amoCRM или Битрикс24), чтобы данные передавались между ними автоматически: контакты, сделки, менеджеры.

## Где её открыть

В левом меню нажмите **«Интеграции»** (значок розетки).

Раздел доступен владельцу и администратору.

## Два режима подключения

Сверху — переключатель:

- **Рекомендуемый: приложение** — «чистая» установка через приложение в маркетплейсе amoCRM/Битрикс24. Клиент нажимает кнопку, подтверждает доступ — всё готово.
- **Быстрый старт: webhook** — если у вас уже есть ссылка-webhook от внешней CRM и вы хотите просто принимать от неё события.

Первый вариант надёжнее и рекомендуется.

---

## Вариант «Приложение»

### Пошагово

1. Выберите **«Рекомендуемый: приложение»**.
2. В поле **«Название подключения»** впишите, как оно будет называться у вас (например, «Основная интеграция»).
3. Нажмите нужную кнопку:
   - **«Установить amoCRM приложение»**.
   - **«Установить Битрикс24 приложение»**.
4. Вас перекинет в страницу вашей CRM, где нужно **подтвердить разрешения** (права на доступ к контактам, сделкам и т.п.).
5. После подтверждения вас вернёт обратно — появится строка подключения.

Рядом есть текстовые ссылки **«OAuth без маркетплейса»** — это альтернативный путь, если у вас уже настроено собственное приложение.

---

## Вариант «Webhook»

### Пошагово

1. Выберите **«Быстрый старт: webhook»**.
2. В поле **«CRM»** выберите amoCRM или Битрикс24.
3. В поле **«Название подключения»** — как его назвать.
4. Заполните поля в зависимости от выбранной CRM.

### Поля для amoCRM
- **Base URL** — адрес вашего аккаунта (например, `https://example.amocrm.ru`).
- **Subdomain (опц.)** — поддомен, если требуется отдельно.
- **Access Token** — токен, который вы получили в настройках amoCRM.
- **Webhook HMAC Secret (опц.)** — секретный код для проверки подлинности запросов. Можно оставить пустым.

### Поля для Битрикс24
- **Webhook URL** — полный адрес входящего webhook (например, `https://portal.bitrix24.ru/rest/1/xyz/`).
- **Base URL (опц.)** — адрес вашего портала.
- **Access Token (опц.)**, **User ID (опц.)**, **Application Token (опц.)** — используются только при некоторых способах авторизации.

5. Нажмите **«Создать webhook-подключение»**.

## Таблица подключений

Все ваши интеграции собраны в таблице. Там видно:

- **Название**, **CRM**, **Режим** — как подключено (Маркетплейс / OAuth / Webhook).
- **Статус** — цветная метка:
  - Зелёная — **работает**.
  - Жёлтая — **нужна доавторизация** или **недостаточно прав**.
  - Красная — **ошибка**.
- **Scope** — если не хватает прав, здесь написано, каких.
- **Webhook URL** — адрес, куда нужно слать события из внешней CRM (показывается для справки).
- **Последняя активность** — когда была синхронизация, проверка, последнее событие.

### Действия со строкой

- ✅ **«Проверить»** — запустить проверку работоспособности. Под таблицей появится результат.
- 🔄 **«Синхронизировать менеджеров»** — подтянуть список менеджеров из внешней CRM.
- 🔁 **«Переавторизовать»** — если срок авторизации истёк.
- ❗ **«Журнал ошибок»** — открыть список последних ошибок (на значке есть цифра — сколько ошибок).
- 🗑️ **«Удалить»** — удалить подключение.

---

## Журнал ошибок

Окно **«Журнал ошибок интеграции»** показывает, что пошло не так:

- Метка **INFO / WARNING / ERROR**.
- Название и подробности ошибки.
- Строка **«Что сделать»** — подсказка, как починить (например, «переавторизуйте подключение»).

---

## Типичные ситуации

- **Первое подключение amoCRM** — режим «Приложение», введите название, нажмите «Установить amoCRM приложение», подтвердите доступ в окне amoCRM.
- **Поменяли пароль во внешней CRM — интеграция отвалилась** — в строке подключения нажмите 🔁 **«Переавторизовать»**.
- **Новые менеджеры не видны у нас** — нажмите 🔄 **«Синхронизировать менеджеров»**.
- **Хотим удалить старое подключение** — нажмите 🗑️ и подтвердите.
- **Нужно понять, почему не работает** — откройте **«Журнал ошибок»** (значок ❗).

## Если не получается

- **Раздел недоступен** — доступ только у владельца и администратора.
- **После нажатия «Установить» ошибка** — проверьте, что вы залогинены в нужной внешней CRM в том же браузере.
- **Статус жёлтый — «Не хватает прав»** — в «Журнале ошибок» будет список отсутствующих прав. Нажмите 🔁 **«Переавторизовать»** и при входе во внешнюю CRM разрешите всё.
- **Статус красный** — откройте журнал ошибок и следуйте подсказкам «Что сделать».
- **Webhook не приходит** — скопируйте адрес из столбца **«Webhook URL»** и проверьте, что он вставлен в настройки вашей внешней CRM.
