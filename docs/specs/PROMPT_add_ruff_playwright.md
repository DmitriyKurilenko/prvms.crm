# Переносимый промт: добавить гейты валидации ruff + Playwright в Docker-проект

> Назначение: вставлять этот промт в другой проект (Django + Vue SPA в Docker или аналог),
> чтобы агент добавил локально запускаемый статанализ (ruff) и сквозную браузерную проверку
> (Playwright) как обязательные гейты. Сформулирован по факту реального внедрения (DEC-048).

---

## Промт (копировать целиком)

Добавь в этот проект два локально запускаемых гейта валидации и сделай их частью ритуала
проверки. Не клади dev-инструменты в production-образ. Перед интеграцией прочитай реальные
конфиги проекта (Dockerfile, docker-compose, requirements/ package.json, CI) и опирайся на
них, а не на догадки.

### 1. ruff (статанализ Python), без загрязнения прод-образа
- Если ruff уже пиннится в `requirements-dev.txt` — используй ту же версию; иначе закрепи версию там.
- Создай отдельный `Dockerfile.dev` (FROM того же python-базового образа), который ставит
  только `requirements-dev.txt` и по умолчанию запускает `ruff check .`.
- Добавь в `docker-compose.yml` сервис `lint` под профилем `tools` (чтобы он не поднимался
  обычным `up`), монтирующий код в `/app`, с `command: ruff check .`.
- Запуск: `docker compose run --rm lint`. Конфиг ruff бери из `pyproject.toml` (тот же, что в CI).
- Прогони и доведи до «All checks passed!». Реальные находки — чини в коде, не подавляй вслепую.

### 2. Playwright (сквозная проверка SPA в реальном браузере)
- Узнай последнюю стабильную версию `@playwright/test` (registry.npmjs.org) и пиннь её в
  `devDependencies` фронтенда; **тег официального Docker-образа обязан совпадать с версией
  пакета**: `mcr.microsoft.com/playwright:vX.Y.Z-jammy`.
- Добавь `playwright.config.ts` (`testDir: './e2e'`, `use.baseURL` из env, `webServer` =
  `npm run build && npm run preview`, один проект chromium) и npm-скрипт `test:e2e`.
- **Same-origin контур (критично).** Собери SPA с относительным `VITE_API_URL=/api` и настрой
  `vite preview.proxy`, чтобы `/api` и `/ws` проксировались на backend-сервис внутри
  compose-сети (`http://web:8000`). Так нет CORS и проблем с `SameSite`-cookie между SPA и API.
- Добавь сервис `e2e` (профиль `tools`) на официальном образе Playwright, монтирующий фронт
  и отдельный том под `node_modules`; команда — `npm install && npx playwright test`.
- **Детерминированные данные без ручных шагов.** Добавь одноразовый сервис `seed`
  (на backend-образе), который сидит тестового пользователя/тенант идемпотентной командой,
  и сделай `e2e` зависимым от `seed` (`condition: service_completed_successfully`) и от `web`.
  Тогда `docker compose run --rm e2e` самодостаточен.

### Грабли, которые надо обойти (проверено на практике)
- **In-memory access-токен теряется при `page.goto`.** Полная перезагрузка страницы разлогинивает
  SPA, и guard уводит на `/login`. Между разделами навигируй кликами по меню (клиентский роутинг),
  а не `page.goto` на защищённый путь. Сам вход делай через форму `/login`.
- **Свежий тенант открывает онбординг.** Проверь, не перехватывает ли мастер настройки работу;
  если да — пройди/пропусти его в начале сценария (учти возможный диалог подтверждения).
- **Диагностируй падения по артефактам, а не перебором.** Читай ARIA-снимок Playwright из
  `test-results/<...>/error-context.md` и исходник компонентов; делай прицельный фикт по факту.
- **Селекторы по доступному имени/тексту**, а не по случайным классам, где это возможно
  (`getByRole`, `getByPlaceholder`, `getByText`).

### Ритуал
- Внеси оба шага (`docker compose run --rm lint` и `docker compose run --rm e2e`) в
  Validation Baseline проекта (`AGENTS.md`/README) как обязательные: ruff — при изменениях
  Python, e2e — при изменениях UI. «Зелёные unit-тесты» без ruff и e2e не считаются «работает».
- Артефакты прогона (`test-results`, `playwright-report`, `blob-report`) добавь в `.gitignore`.

### Критерий приёмки
- `docker compose run --rm lint` → «All checks passed!».
- `docker compose run --rm e2e` → как минимум один сквозной сценарий `passed` в реальном
  headless-браузере (вход → ключевой пользовательский поток → проверка результата на экране).
